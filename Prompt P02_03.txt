
judul: Sistem Manajemen Pesanan Kafetaria IT Del
Bertindaklah sebagai Senior C Developer dan Konsultan Sistem Informasi.
Tujuan: Membangun sistem pencatatan pesanan kafetaria yang efisien untuk mengelola data menu dan total harga secara otomatis.
PILAR 1: KEJELASAN
Masalah: Kafetaria memiliki daftar menu (Contoh: Nasi Goreng: 15000, Ayam Penyet: 20000, Teh Manis: 5000). Data ini tersusun secara linear.
Logika: Gunakan pendekatan Chain of Thought. Jelaskan bagaimana program memetakan nomor menu yang dipilih pengguna (1 sampai N) ke dalam indeks array 0 sampai n-1.
Fungsi: Program harus bisa menghitung total harga berdasarkan jumlah porsi yang dipesan untuk satu kategori menu tertentu.
 PILAR 2: KONTEKSTUALISASI 
Lingkungan: C Standard (C11), dikompilasi dengan gcc di terminal.
Alur Input:
Input N (Jumlah jenis menu yang tersedia).
Input Deret Data (Nama Menu dan Harga per porsi).
Input Kode Menu yang dipesan dan Jumlah Porsi.
 PILAR 3: STRUKTUR & FORMAT
Modularitas: Pisahkan logika perhitungan ke dalam fungsi hitungTotalPesanan.
Output Struk: Tampilkan hasil akhir dalam format struk ringkas:
[Kafetaria IT Del] | Menu: [Nama] | Porsi: [Qty] | Total: Rp [Harga].
Dokumentasi: Berikan komentar // penjelasan logika pada setiap baris kode.
PILAR 4: BATASAN 
Library: Hanya gunakan <stdio.h> dan <string.h>.
Memori: Wajib menggunakan Array Statis (maksimal 50 menu). Hindari malloc.
Error Handling: Jika kode menu < 1 atau > N, tampilkan pesan "Menu Tidak Tersedia!".
Exit Condition: Program berhenti jika $N = 0$.
Gunakan bahasa manusia yang lugas, teknis, dan hindari sapaan robotik yang tidak perlu.
